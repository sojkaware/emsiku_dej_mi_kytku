# Support & FAQ

## Q&A

### Got 403 while downloading models or images?

For security reasons, the download link is valid for 60 seconds. You need to download the file within this time frame. If the link expires, you will need to request a new download link from task.

### What is the relation between webapp and platform?

Please note that our webapp and API operate under two independent billing systems. The credits for each system are not shared between them. Additionally, the pricing structures for the webapp and API differ. Each has its own top-up entry, so you will need to recharge them separately.

### Got 401 for every requests?

Please note that your API key and Client ID are not the same. The API key starts with “tsk_” while the Client ID starts with “tcli_”. Each serves a distinct purpose and should be used correctly.

- API Key: This is a unique identifier used to authenticate requests associated with your project for usage and billing purposes. It is essential to use the correct API key to ensure your requests are properly authorized.
- Client ID: This is used to identify your application or service when interacting with the API. It is different from the API key and is not used for authentication.

For your API calls to work properly, make sure you are using the correct API key. If you use the Client ID instead of the API key, your requests will not be authenticated and will fail.
